## Цель

Перевести раздел «По магазинам» с кассовых данных (`filteredRows`) на начислительные данные из «ДДС Гульжан А.» (`buildGulzhanData()`). Добавить визуальные маркеры сдвигов и статус «ожидает оплаты» для неоплаченных статей.

## Источник данных

### Было
`renderStoreGrid()` вручную агрегирует `filteredRows` в `storeMap{inc, out, groups, rentArticles}`.

### Станет
```js
const { stores, groups, totalIncome } = buildGulzhanData(currentMonth);
```

Транспонирование в per-store данные:
```
для каждого store:
  salesTotal = groups['Продажи'].subtotals[store]          // поступления
  displayOut = |Σ expense-групп .subtotals[store]|         // выбытия
  cf         = salesTotal - displayOut
  pct        = displayOut / salesTotal * 100
```

Загрузка смежных месяцев (как в `renderGulzhanMatrix()`):
```js
ensureMonthLoaded(currentMonth - 1);
ensureMonthLoaded(currentMonth + 1);
// + квартальные месяцы
```

**Компания/Офис** — без изменений: используют отдельный путь с `filteredRows`.

## Логика shift-лейблов

### Для аренда-статей (индивидуальные строки)

| Статья | Тип | Лейбл |
|---|---|---|
| Аренда торговых точек, Аренда офиса, Аренда и Охрана Склада | advance | `▲ данные за [M-1]` |
| Ком услуги, Охрана, Обеспечительный взнос | — | — |
| Аренда переменная (Астана, Above Астана) | prevmonth | `▼ данные за [M+1]` |
| Аренда переменная (Есентай) | quarterly | `◇ квартальные` |
| Аренда переменная (остальные магазины) | — | — |

### Для групп (одна строка)

```
getGroupShiftType(groupName):
  собираем shift-типы всех статей в группе
  все prevmonth           → 'prevmonth'  → ▼
  все quarterly           → 'quarterly'  → ◇
  mix prevmonth+quarterly → 'mixed'      → ◆
  иначе                   → null         → без лейбла
```

Примеры:
- **Зарплаты** (все prevmonth) → `▼ данные за [M+1]`
- **Налоги** (Налоги ЗП prevmonth + Налоги ИП quarterly + Налоги текущий) → `◆ сдвиг по части`
- **Маркетинг** (все текущий) → без лейбла

CSS-классы те же что в Гульжан-матрице: `gul-advance`, `gul-prevmonth`, `gul-quarterly`, `gul-mixed`.

## Логика «ожидает оплаты»

### Правило

| Тип статьи | Условие "ожидает" |
|---|---|
| advance (аренда M-1) | **никогда** — всегда исторические данные |
| prevmonth | `amount[store] === 0` |
| quarterly | `amount[store] === 0` |
| текущий месяц | **никогда** — 0 означает отсутствие расхода |

### Для аренда-строк (индивидуальные)
Если статья — prevmonth/quarterly И `byStore[store] === 0` → вместо суммы и `%` показываем `⏳ ожидает оплаты`.

### Для группных строк
Если `subtotals[store] === 0` И в группе есть хотя бы одна prevmonth/quarterly статья → вместо суммы и `%` показываем `⏳ ожидает оплаты`.
