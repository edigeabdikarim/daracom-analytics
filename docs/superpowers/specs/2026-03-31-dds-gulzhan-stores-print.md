## Контекст

Три раздела `dds.html` требуют доработок:
- **ДДС Гульжан А.** — матрица статьи × магазины (начислительный подход)

## Блок 1 — ДДС Гульжан А.: логические правки матрицы

### 1.1 «Ком услуги» → prevmonth-сдвиг

Добавить `'Ком услуги'` в константу `GULZHAN_PREV_MONTH_ARTICLES`.

**Эффект:**
- В матрице статья получит метку `▼ [M+1]` и синюю подсветку (`gul-prevmonth`)
- В карточках «ДДС Магазины Гульжан А.» автоматически заработает «⏳ ожидает оплаты» при нулевом значении (через существующую логику `getArticleShiftType`)

### 1.2 «Ожидает оплаты» в ячейках матрицы

Для определённых статей/магазинов пустая ячейка означает «платёж ещё не проведён».  
Вместо `0` показывать `⏳` в конкретной ячейке `(статья, магазин)`.

**Функция `isCellAlwaysPaid(articleName, storeName) → boolean`:**
```
1. GULZHAN_PREV_MONTH_ARTICLES.has(articleName)          → true (все магазины)
2. GULZHAN_PREVMONTH_STORES[articleName]?.has(storeName) → true (конкретные магазины)
3. GULZHAN_QUARTERLY_STORES[articleName] === null        → true (все магазины)
4. GULZHAN_QUARTERLY_STORES[articleName]?.has(storeName) → true (конкретные магазины)
```

Затронутые статьи после добавления «Ком услуги»:

| Статья | Магазины |
|---|---|
| Зарплата Продавцы | все |
| Зарплата Офис | все |
| Зарплата Аутсорс | все |
| Зарплата Техперсонал | все |
| Зарплата СММ маркетинг, мерчандайзер | все |
| Зарплата Водитель | все |
| Налоги ЗП | все |
| Ком услуги | все (после 1.1) |
| Аренда переменная | Магазин Астана, Магазин Above Астана |
| Аренда переменная | Магазин Есентай (quarterly) |
| Налоги ИП (квартальные) | все |

**Рендер ячейки в `renderGulzhanMatrix`:**
```js
if (isCellAlwaysPaid(a.name, s) && Math.abs(a.byStore[s] || 0) < 0.01) {
  html += `<td style="color:#9CA3AF;text-align:center;font-size:11px;">⏳</td>`;
} else {
  html += `<td>${renderSummaryCell(a.byStore[s] || 0)}</td>`;
}
```

**Строки «always-paid» статей не скрываются** при «Скрыть нулевые»:  
Строка считается «нулевой» (`data-zero="1"`) только если `isZero && !rowHasAlwaysPaidCell`, где  
`rowHasAlwaysPaidCell = stores.some(s => isCellAlwaysPaid(a.name, s))`.

### 1.3 Удалить колонку «Общее»

В `buildGulzhanData`, при формировании списка `stores`, добавить фильтр:
```js
.filter(s => !knownStores.has(s) && s !== 'Компания' && s !== 'Общее')
```
Транзакции без магазина (`r.store` пустой) продолжают учитываться в `cellData`, но колонка «Общее» не попадает в список отображаемых магазинов.

### 1.4 Удалить группу «Прочее»

В `buildGulzhanData` убрать блок создания группы «Прочее»:
```js
// Удалить (строки 1527-1532):
const extraArticles = ...
if (extraArticles.length > 0) {
  groupsMap.set('Прочее', ...)
}
```
Статьи не из `DDS_ARTICLES_ORDER` тихо игнорируются.

## Блок 2 — Печать: ДДС Гульжан А.

### 2.1 Убрать чипы остатков из print-шапки

`#gulzhanPrintHeader` содержит элемент `#gulzhanPrintBalances` — он заполняется карточками остатков в `renderGulzhanMatrix()`. Перестать его наполнять (оставить пустым).

В обычном виде `#gulzhanBalanceChips` остаётся без изменений.

### 2.2 Месяц в print-шапке

В `#gulzhanPrintHeader` уже есть `<span class="ph-month"></span>` — месяц подставляется туда. Убедиться, что `renderGulzhanMatrix()` обновляет все `.ph-month` внутри секции (уже реализовано через `querySelectorAll('.ph-month')`).

### 2.3 Повторяющаяся шапка на каждой печатной странице

Реализация через CSS `position: fixed` + `@media print`:

```css
@media print {
  .print-repeat-header {
    position: fixed;
    top: 0; left: 0; right: 0;
    background: #fff;
    border-bottom: 2px solid #1E3A5F;
    padding: 8px 16px;
    z-index: 9999;
  }
  body { margin-top: 60px; } /* отступ под шапку */
}
```

Шапка содержит: логотип/название «ДДС · Daracom — Гульжан А.» + название месяца.

### 2.4 Нумерация страниц

```css
@media print {
  .print-page-counter {
    position: fixed;
    bottom: 8px; right: 16px;
    font-size: 10px; color: #9CA3AF;
  }
  .print-page-counter::after {
    content: "Стр. " counter(page);
  }
}
```
Элемент `.print-page-counter` добавляется один раз в DOM (виден только при печати).

### 4.2 Группа «Аренда и ком. услуги» — заголовок + вложенные подстатьи

**Текущая структура** в карточке: плоский список статей аренды (`rentRows`).

**Новая структура:**
```
┌─ Аренда и ком. услуги  [итог]  XX%    ← строка-заголовок группы
│    Аренда торговых точек ▲ ...  XXX тыс.  XX%
│    Ком услуги ▼ ...             XXX тыс.  XX%
│    Аренда переменная ◇ ...      ⏳ ожидает оплаты
│    ...
```

Строка-заголовок:
- Жирный текст, цвет `#374151`, фон `#F9FAFB`
- Сумма = `Math.abs(Σ rentRows[].amt)` (включая pending-строки по сумме 0)
- `%` от продаж
- Нулевые статьи без pending → скрыты (как раньше)

Подстатьи:
- `padding-left: 16px` для визуального отступа
- цвет текста `#6B7280` (чуть светлее, чем заголовок)

Если сумма группы = 0 и ни одна подстатья не pending → строка-заголовок скрыта.

### 4.3 Группа «Налоги» — заголовок + вложенные подстатьи

Аналогично 4.2 для группы «Налоги».

Подстатьи (из `taxGroup.articles` через `buildGulzhanData`):
- `Налоги ЗП` — prevmonth → ⏳ если 0
- `Налоги ИП (квартальные)` — quarterly → ⏳ если 0
- `Налоги` — обычная статья

Нулевые обычные статьи → скрыты. Pending → показываются с ⏳.

**В `sortedExpGroups`** (отфильтровывает «Аренда и ком. услуги» для `otherRows`) добавить исключение «Налоги»:
```js
.filter(g => !EXCLUDED_GROUPS.has(g.name)
          && g.name !== 'Аренда и ком. услуги'
          && g.name !== 'Налоги'        // ← добавить
          && g.isIncome === false)
```
«Налоги» рендерятся отдельным блоком с раскрытыми подстатьями, аналогично аренде.

### 4.4 Печать: убрать остатки + месяц + шапка

## Архитектурные детали

### Подход к повторяющимся шапкам

Каждая секция уже имеет свой print-header (`#gulzhanPrintHeader`, `#summaryPrintHeader`, `#storesPrintHeader`). В `@media print` каждый из них получает `position: fixed; top: 0` — браузер повторяет fixed-элементы на каждой странице.

Только активная секция видна при печати (`@media print .section:not(.active) { display: none }`), поэтому конфликта шапок нет.

Нумерация страниц — один глобальный элемент:
```html
<div class="print-page-counter"></div>
```

### CSS @media print (общий блок)

```css
@media print {
  /* Скрыть навигацию, KPI, sidebar и прочее */
  .sidebar, .nav-bar, .kpi-grid, #filterBar { display: none !important; }

  /* Шапка */
  .print-repeat-header { display: block !important; }

  /* Нумерация */
  .print-page-counter { display: block !important; }

  /* Отступ под шапку */
  .section.active { margin-top: 56px; }
}
```
