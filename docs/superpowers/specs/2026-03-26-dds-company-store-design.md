# Spec: Отображение статей «Компания» в Сводной матрице ДДС

**Дата:** 2026-03-26
**Файл:** `dds.html` (только)
**GAS:** не затрагивается

---

## Контекст

В данных ДДС появилось новое значение поля `store` — `"Компания"`. Оно означает, что операция относится ко всей компании целиком и не может быть привязана к конкретному магазину. Статьи с таким store не имеют транзакций в других магазинах (взаимоисключающее условие).

Текущая проблема: `"Компания"` попадает в массив `stores` и отображается как обычная колонка в матрице статей × магазины, из-за чего все магазинные ячейки у таких статей — нули, а значение сосредоточено в одной колонке «Компания». Это визуально запутывает.

---

## Решение

### Поведение в матрице

Для статей, у которых **все транзакции** имеют `store = "Компания"` (флаг `isCompanyWide = true`):

- Колонка «Итого» — показывает сумму в обычном режиме (без изменений)
- Все колонки магазинов — объединяются в одну ячейку (`colspan = N`, где N = кол-во магазинов), значение = то же что в «Итого», выравнивание по центру

Для остальных статей — поведение без изменений.

Для строк-заголовков групп (`smg-row`) — поведение без изменений; subtotals по магазинам считаются только по обычным статьям.

### Пример

```
Обычная статья:
┌─────────────────────┬───────────┬──────────┬──────────┬──────────┐
│ Зарплата продавцов  │  500 000  │  150 000 │  100 000 │   80 000 │ ...
└─────────────────────┴───────────┴──────────┴──────────┴──────────┘

Статья «Компания»:
┌─────────────────────┬───────────┬──────────────────────────────────┐
│ Закупка товара      │18 090 000 │         18 090 000                │
│                     │           │  ← colspan = N магазинов →        │
└─────────────────────┴───────────┴──────────────────────────────────┘
```

---

## Изменения в коде

### `buildSummaryData(rows)`

1. **Исключить `"Компания"` из `stores`:**
   После сортировки магазинов по обороту — отфильтровать `"Компания"` из результирующего массива `stores`.

2. **Флаг `isCompanyWide` для статей:**
   При формировании статей проверять: если `cellData[article]` содержит только ключ `"Компания"` (или все остальные магазины имеют нулевое значение), ставить `isCompanyWide: true`. Иначе `isCompanyWide: false`.

### `renderSummaryMatrix()`

В блоке рендера строк статей (`g.articles.forEach`):

- Если `a.isCompanyWide === true`:
  ```html
  <td colspan="${stores.length}" style="text-align:center">
    ${renderSummaryCell(a.total)}
  </td>
  ```
- Если `a.isCompanyWide === false` — рендер без изменений (по одной `<td>` на каждый store).

---

## Что НЕ меняется

- Toggle «Скрыть нулевые» — работает как прежде (проверка `a.total`)
- CSS `.summary-tbl` — без изменений
- GAS-скрипты — не затрагиваются
- Остальные секции `dds.html` — не затрагиваются

---

## Граничные случаи

| Случай | Поведение |
|--------|-----------|
| Статья имеет данные и в «Компания», и в магазинах | `isCompanyWide = false`, отображается как обычная статья (магазин «Компания» игнорируется в колонках, его сумма войдёт только в «Итого») |
| `stores` пуст после фильтрации | Матрица не рендерится (существующая проверка) |
| Новые значения store в будущем | Фильтрация только `"Компания"`, остальные добавляются автоматически |
