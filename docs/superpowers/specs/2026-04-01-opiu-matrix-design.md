## Правила отображения

### Company-wide строки
- Ячейки направлений объединяются: `colspan="9"`
- Значение (из колонки Итого) отображается по центру объединённой ячейки
- % от общей выручки показывается под значением

### Per-direction строки
- Каждое направление — отдельная ячейка
- % от выручки соответствующего направления под значением

### Расчётные строки [CALC]
- Значения берутся из GAS-данных (уже вычислены в листе)
- Если строки нет в GAS — вычислить в JS по формуле
- `% от выручки` показывается (база = общая выручка для company-wide, выручка направления для per-direction)

### L1-группа Итого
- Производственные расходы: `Переменные.total + Прямые_постоянные.total + Общепроизводственные.total`
- Производственные расходы по направлениям: `Переменные[dir] + Прямые_постоянные[dir]` (Общепроизводственные не имеет per-direction)
- Косвенные расходы: company-wide, только Итого

### Поток данных

1. Fetch `?action=opiu&month=N` — GAS возвращает `{ month, stores[], rows[] }`
2. Индексировать: `rowByLabel = {}` — O(1) lookup
3. Найти `revenueRow` для базы %
4. Итерировать по `data.rows` в GAS-порядке → для каждой строки определить тип через `OPIU_ROW_CONFIG`
5. Для L1-групп: если строка есть в GAS — использовать значение; если нет — вычислить из L2-сумм
6. Рендерить `renderOpiuMatrix()`

### Ключевые функции

- `classifyRow(label)` → `'l0' | 'l1' | 'l2' | 'calc' | 'article'`
- `isCompanyWide(label)` → boolean
- `renderOpiuRow(row, stores, totalRevenue, storeRevenue)` → HTML строка
- `buildOpiuCell(val, revenue, isCompanyWide)` → HTML ячейка с % под значением
- `computeL1Total(l1label, rowByLabel, stores)` → `{ total, byStore }`
