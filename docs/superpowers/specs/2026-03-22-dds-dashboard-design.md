## Архитектура

### Файлы

| Файл | Назначение |
|------|-----------|
| `dds.html` | Фронтенд дашборда (vanilla JS + Tailwind + Chart.js) |
| `gas/dds.gs` | GAS скрипт — новый endpoint, возвращает все строки за выбранный месяц |

### GAS Endpoint (`dds.gs`)

- Параметр: `?month=3`
- Возвращает **все строки** за месяц (не агрегирует на сервере)
- Клиент сам группирует, фильтрует, считает
- Деплой: новый Web App deployment, доступ Anyone
- URL хранится в константе `DDS_GAS_URL` в начале `dds.html` (аналогично `CONFIG.GAS_URL` в `dashboard.html`)

**Структура ответа:**
```json
{
  "ok": true,
  "data": {
    "month": 3,
    "rows": [
      {
        "date": "01.03.2026",
        "amount": -3459200,
        "wallet": "ИП Базарханова б/н",
        "store": "Магазин Абайка",
        "counterparty": "",
        "purpose": "Аренда торговых точек",
        "article": "Аренда торговых точек",
        "type": "Выбытие",
        "direction": "Адм. расходы",
        "activity": "Операционная"
      }
    ]
  }
}
```

### Выбор месяца (Topbar)

- По умолчанию — текущий месяц (`new Date().getMonth() + 1`)
- Список месяцев: Январь–Декабрь текущего года (hardcoded 2026)
- При смене месяца — перезапрос к GAS (с учётом кэша)
- Кэш-ключ: `dds_cache_month_{N}` (включает номер месяца)

### Layout: адаптивность

- ≥ 1200px: сайдбар 200px + основная область
- < 1200px (планшет): сайдбар сворачивается, показывается кнопка-гамбургер ☰
- < 768px: не поддерживается (дашборд для десктопа)

### Состояния загрузки и ошибок

- **Loading:** спиннер по центру основной области, сайдбар заблокирован
- **Error:** карточка с текстом ошибки и кнопкой «Повторить»
- **Empty (нет данных за месяц):** заглушка «Нет данных за [месяц]»
- **Empty (фильтр дал 0 строк):** заглушка «Нет операций по выбранным фильтрам»

### Раздел 1: Обзор

**KPI карточки (4 шт.):**
- Поступления за месяц (зелёный)
- Выбытия за месяц (красный)
- Чистый Cash Flow = Поступления − Выбытия (жёлтый/зелёный)
- Операционные расходы (расходы с `Вид д-ти = Операционная`, `Платеж = Выбытие`, `Направл ≠ Операция`)

**Топ-5 статей расходов** — горизонтальный барчарт, кликабельный.

### Раздел 2: Куда ушли деньги

Полный список статей расходов с суммами и процентами от общего.
Сортировка: по убыванию суммы. Переключение сортировки — кнопки «По сумме» / «По алфавиту» над списком.
Клик на строку → выезжает правая панель с операциями по этой статье.

### Раздел 3: По магазинам

Сетка карточек (3 колонки) по каждому магазину + Офис:
- Поступления
- Выбытия
- Чистый CF (цветной)

Клик на карточку → правая панель с операциями по этому магазину.

### Раздел 4: Деньги по дням

Столбчатый график (Chart.js): по X — дни месяца, по Y — суммы.
Два ряда: Поступления (зелёный) и Выбытия (красный).
Под графиком — таблица топ-10 крупнейших операций за месяц.

### Правая панель (Drill-down)

Появляется при клике на статью или магазин. Закрывается кнопкой ✕ или кликом на оверлей.

**При клике на статью (раздел «Куда ушли деньги»):**
- Заголовок: название статьи
- Сводная сумма + количество операций
- Таблица: Дата | Магазин | Назначение | Сумма

**При клике на магазин (раздел «По магазинам»):**
- Заголовок: название магазина
- Поступления / Выбытия / Чистый CF
- Таблица: Дата | Статья | Назначение | Тип | Сумма

## Логика расчётов (клиентская, JS)

Поле `amount` в ответе GAS: **отрицательное для Выбытий, положительное для Поступлений**.
JS всегда использует `Math.abs(row.amount)` при суммировании расходов.

Точные строковые значения (из реальных данных таблицы):
- `type`: `'Поступление'` / `'Выбытие'`
- `activity`: `'Операционная'` / `'Техническая операция'` / `'Инвестиционная'`
- `direction`: `'Продажи'` / `'Зарплаты'` / `'Адм. расходы'` / `'Налоги'` / `'ОС'` / `'Операция'` / `'Платежи'`

```
Поступления    = SUM(rows: type==='Поступление' AND activity!=='Техническая операция')
Выбытия        = SUM(rows: type==='Выбытие' AND activity!=='Техническая операция', Math.abs)
Чистый CF      = Поступления − Выбытия
Опер. расходы  = SUM(rows: type==='Выбытие' AND activity==='Операционная' AND direction!=='Операция', Math.abs)

По статьям     = группировка Опер. расходов по полю article, Math.abs
По магазинам   = группировка всех строк (activity!=='Техническая операция') по полю store
По дням        = группировка (activity!=='Техническая операция') по дню из date
Кэш-ключ       = 'dds_cache_month_' + month (отдельный кэш на каждый месяц)
```

## Технический стек

- **Frontend:** HTML + Tailwind CSS (CDN) + Chart.js 4 + vanilla JS (без фреймворков)
- **Backend:** Google Apps Script Web App
- **Кэш:** `localStorage`, TTL 30 минут (аналогично `dashboard.html`)
- **Деплой фронта:** GitHub Pages (рядом с `dashboard.html`)

---

## Связь с существующим дашбордом

- `dashboard.html` остаётся без изменений
- `dds.html` — новая отдельная страница
- `gas/dds.gs` — новый GAS файл (не меняем `dashboard.gs`)
- Навигация: в `dashboard.html` добавить ссылку «ДДС →» в header

---

## Критерии готовности

1. Данные за выбранный месяц загружаются из GAS за < 5 секунд
2. Все 4 раздела навигации работают
3. Правая панель открывается при клике на статью и на магазин
4. Фильтры применяются ко всем разделам
5. Числа совпадают с данными в Google Sheets (проверяется вручную руководителем)
6. Светлая тема, адаптивный layout для широких экранов (≥ 1200px)
