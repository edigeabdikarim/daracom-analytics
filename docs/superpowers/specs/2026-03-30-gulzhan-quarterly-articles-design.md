## Контекст

Раздел «ДДС Гульжан А.» показывает расходы по реальному периоду (не по дате платежа). Нужно:
1. Исправить отображение апреля (и любого месяца без транзакций)

## 1. Фикс пустых месяцев (апрель и далее)

**Проблема:** `onDataLoaded()` вызывает `showState('empty')` когда `allRows = []` → `renderGulzhanMatrix()` не вызывается, хотя авансовые данные из M-1 доступны.

**Решение:**
- В `onDataLoaded()`: если секция 'gulzhan' активна (проверка через `document.getElementById('section-gulzhan').classList.contains('active')`), вызывать `showState('content')` + `renderAll()` даже при пустом `allRows`
- В `renderGulzhanMatrix()`: убрать ранний `return` при `allRows.length === 0`; показывать "нет данных" только если `buildGulzhanData()` вернул пустой `stores`

## 3. Логика сдвигов в Gulzhan

### Таблица правил

| Статья | Магазин | Логика | Источник данных |
|---|---|---|---|
| Аренда переменная | Магазин Астана, Магазин Above Астана | prevmonth | allMonthsData[M+1] |
| Аренда переменная | Магазин Есентай | квартальная | расчёт из платёжного месяца |
| Аренда переменная | остальные | текущий месяц | allMonthsData[M] |
| Обеспечительный взнос | все | текущий месяц | allMonthsData[M] |
| Налоги ИП (квартальные) | все | квартальная | расчёт из платёжного месяца |

### Новые константы

```js
// Статьи, где только часть магазинов берётся из M+1
const GULZHAN_PREVMONTH_STORES = {
  'Аренда переменная': new Set(['Магазин Астана', 'Магазин Above Астана']),
};

// Статьи с квартальным распределением (null = все магазины)
const GULZHAN_QUARTERLY_STORES = {
  'Аренда переменная':       new Set(['Магазин Есентай']),
  'Налоги ИП (квартальные)': null,
};
```

## 4. Квартальная логика

### Квартальные периоды

| Квартал | Месяцы | Платёжный месяц |
|---|---|---|
| Q1 | Янв, Фев, Мар | Апрель (4) |
| Q2 | Апр, Май, Июн | Июль (7) |
| Q3 | Июл, Авг, Сен | Октябрь (10) |
| Q4 | Окт, Ноя, Дек | Январь (13 → не реализуем) |

### Формула распределения

Для месяца M, магазина S, платёжного месяца P:

```
allocation[S][M] = payment_S_in_P × (sales_S_in_M / total_sales_S_in_quarter)
```

Где `sales` = строки с `article === 'Продажи товаров'` по данному магазину.

Если `total_sales_S_in_quarter === 0` → показывать `—` (нет продаж для пропорции).

### Новая функция

```
buildQuarterlyRows(article, storeSet, month) → Row[]
```

- Определяет квартал и платёжный месяц
- Загружает данные через `ensureMonthLoaded()` для всех 3 месяцев квартала + платёжного месяца
- Берёт платёжные строки из `allMonthsData[paymentMonth]` (фильтр по storeSet если не null)
- Считает продажи по магазинам за квартал
- Возвращает синтетические строки с пропорциональной суммой

## 5. Рефакторинг buildGulzhanData()

### Новый порядок сборки строк

```
rowsCurrent:
  - все строки из allMonthsData[M]
  - исключить: GULZHAN_ADVANCE_ARTICLES
  - исключить: GULZHAN_PREV_MONTH_ARTICLES
  - исключить: строки где (article ∈ GULZHAN_PREVMONTH_STORES) && (store ∈ storeSet)
  - исключить: строки где (article ∈ GULZHAN_QUARTERLY_STORES) && (storeSet === null || store ∈ storeSet)

rowsAdvance:      allMonthsData[M-1] → GULZHAN_ADVANCE_ARTICLES (без изменений)
rowsPrevMonth:    allMonthsData[M+1] → GULZHAN_PREV_MONTH_ARTICLES (без изменений)
rowsPrevMonthPartial: allMonthsData[M+1] → per-article по GULZHAN_PREVMONTH_STORES
rowsQuarterly:    buildQuarterlyRows() для каждой статьи в GULZHAN_QUARTERLY_STORES
```

## 6. Обновление renderGulzhanMatrix()

### Загрузка соседних месяцев

Добавить вызовы `ensureMonthLoaded()`:
- Для платёжного месяца квартала: `ensureMonthLoaded(quarterPaymentMonth(currentMonth))`
- Для всех 3 месяцев квартала (нужны продажи): `qMonths.forEach(m => ensureMonthLoaded(m))`

### Визуальные маркеры

| CSS-класс | Цвет | Лейбл |
|---|---|---|
| `.gul-advance` | янтарный `#FEF3C7 / #92400E` | `▲ данные за [пред. месяц]` (без изменений) |
| `.gul-prevmonth` | сиреневый `#EDE9FE / #5B21B6` | `▼ данные за [след. месяц]` (без изменений) |
| `.gul-quarterly` | бирюзовый `#CFFAFE / #155E75` | `◇ квартальные пропорции` |
| `.gul-mixed` | серо-синий `#E0F2FE / #075985` | `◆ сдвиг по части магазинов` |

«Аренда переменная» — получает класс `gul-mixed` (т.к. разные магазины имеют разную логику).
«Налоги ИП (квартальные)» — получает класс `gul-quarterly`.

## 7. Edge cases

- **Платёжный месяц ещё не загружен:** `buildQuarterlyRows` возвращает `[]`, ячейка показывает `—`. При загрузке `_refreshAfterMonthLoad()` перерисует.
- **Нет продаж по магазину в квартале:** пропорция = 0 → не создаём синтетическую строку.
- **Q4 (платёжный месяц = январь следующего года):** возвращаем `[]`, показываем `—`.
- **Январь + авансовая аренда:** GULZHAN_DEC2025_RENT без изменений.
