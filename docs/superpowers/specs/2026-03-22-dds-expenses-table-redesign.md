# ДДС: Редизайн вкладки «По статьям» в блоке «Куда ушли деньги»

**Дата:** 2026-03-22
**Файл:** `dds.html`
**Статус:** Approved

---

## Контекст

Текущая вкладка «По статьям» в блоке «Куда ушли деньги» показывает expandable-список статей с узкими барами (80px) и tooltip только по магазинам. Руководителю неудобно читать — данные «съезжают», непонятна структура расходов по направлениям бизнеса.

---

## Что меняется

### 1. Вкладка «По статьям» → таблица

Заменить expandable-список на таблицу со следующими колонками:

| Колонка | Описание |
|---------|----------|
| **Статья** | Название статьи |
| **Направления** | Мини-тэги направлений (Продажи, Зарплаты, Адм. расходы…), фон `#F3F4F6` |
| **Доля** | Горизонтальный бар 110px. Расходы: градиент `#EF4444→#F97316`. Доходы: `#10B981→#34D399` |
| **Сумма, ₸** | Правое выравнивание. Расходы — `#EF4444`, доходы — `#10B981` |
| **%** | Двустрочная ячейка (см. ниже) |

### 2. Порядок групп

**Поступления — всегда первым блоком**, расходы — вторым.

Заголовок-строка группы:
- Поступления: фон `#F0FDF4`, текст `#059669`
- Расходы: фон `#FEF2F2`, текст `#B91C1C`

### 3. Фиксированный порядок статей в Поступлениях

Константа в начале JS-блока:
```js
const INCOME_ARTICLE_PRIORITY = ['Выручка от продаж', 'Возврат товаров'];
```

Сортировка (точное совпадение строк, case-sensitive, trim):
1. Статьи из `INCOME_ARTICLE_PRIORITY` — в порядке их индекса в массиве
2. Остальные — по убыванию суммы

Функция `sortIncomeArticles(articles)` применяет эту логику. Используется в `renderArticlesTable()` и `renderMatrixTable()`.

### 4. Двустрочная ячейка «%»

```
32%          ← font-size:12px, font-weight:700, color:#374151 — доля в своей группе
от выр: 21%  ← font-size:10px, color:#9CA3AF — % от суммы поступлений
```

- **Расходы**: строка 1 = статья / итого_расходов × 100. Строка 2 = статья / итого_поступлений × 100, подпись «от выр: X%»
- **Поступления**: строка 1 = статья / итого_поступлений × 100 (зелёный `#10B981`). Строка 2 = подпись «от итого» без числа

### 5. Hover-tooltip (два блока)

Механизм передачи данных в tooltip: строки таблицы рендерятся с атрибутами `data-name` и `data-type`. `onmouseenter` читает `event.currentTarget.dataset.name` и `event.currentTarget.dataset.type`. Массив `_artItems` убирается.

Структура tooltip:
```
┌─────────────────────────────────┐
│ ПО НАПРАВЛЕНИЯМ    (9px, серый) │
│  Зарплаты    ████░   2 960 000  │  ← бар: % от макс в блоке
│  Адм. расходы ██░░   1 040 000  │
├─────────────────────────────────┤  ← разделитель rgba(255,255,255,0.08)
│ ПО МАГАЗИНАМ       (9px, серый) │
│  Маг. Астана  ████░    820 000  │
│  ...все магазины...             │
└─────────────────────────────────┘
```

- Мини-бары: ширина = значение / max_в_блоке × 44px
- Цвет сумм: расходы `#F87171`, доходы `#34D399`
- Цвет баров: расходы `#F87171`, доходы `#34D399`
- `max-width` tooltip: увеличить до 260px
- Если один из блоков пуст (нет данных) — не рендерить секцию и разделитель
- `getDirectionBreakdown(rows, name, type)` — новая функция, аналог `getStoreBreakdown()`, группирует по `r.direction`. Использует идентичные фильтры:
  - income: `r.type !== 'Поступление' || r.activity === 'Техническая операция'` → пропустить
  - expense: `r.type !== 'Выбытие' || r.activity !== 'Операционная' || r.direction === 'Операция'` → пропустить
  - Если `r.direction` пустой или null — группировать как `'Прочее'`

### 6. Убрать click-to-expand

Удалить из JS:
- `toggleExpandArticle()`
- `buildExpandBody()`
- переменная `expandedArticles`
- массив `_artItems`

Удалить из CSS:
- `.article-item`, `.article-item:hover`, `.article-item:last-child`
- `.article-name`, `.article-pct`, `.article-amount`, `.article-bar-wrap`, `.article-bar`
- `.expand-chevron`, `.article-item.is-expanded .expand-chevron`
- `.article-expand-body`, `.article-expand-body.open`
- `.mini-bars`, `.mini-bar-col`, `.mini-bar-fill`, `.mini-bar-lbl`, `.mini-bar-val`
- `.article-group-hdr` (заменяется group-row в таблице)

### 7. Убрать кнопки сортировки

В новой таблице порядок статей фиксирован (INCOME_ARTICLE_PRIORITY + по сумме) — кнопки «По сумме» / «По алфавиту» убираются.

Удалить из HTML: блок `#expSortBtns` с кнопками.
Удалить из JS: `setSortArticles()`, переменную `articleSortMode`.

### 8. Обновить все call-sites `renderExpandableList()`

Заменить `renderExpandableList()` на `renderArticlesTable()` везде:

| Место | Строка (ориентировочно) |
|-------|------------------------|
| `renderExpensesSection()` | ~975 |
| `setExpensesTab()` | ~987 |
| `_refreshAfterMonthLoad()` | ~1024 |

В `_refreshAfterMonthLoad()` также удалить проверку `Object.keys(expandedArticles).length` — теперь просто вызывать `renderArticlesTable()` когда `expensesTab === 'list'`.

### 9. Матрица «Таблица» — фиксированный порядок

В `renderMatrixTable()` применить `sortIncomeArticles(incomeArticles)` после формирования массива поступлений (строка после `Object.entries(incomeMap).sort(...)`).

## Новые функции

| Функция | Сигнатура | Назначение |
|---------|-----------|-----------|
| `INCOME_ARTICLE_PRIORITY` | `const string[]` | Фиксированный порядок статей поступлений |
| `sortIncomeArticles(articles)` | `([string, number][]) → [string, number][]` | Сортировка: приоритет → по сумме |
| `renderArticlesTable()` | `() → void` | Рендер таблицы, заменяет `renderExpandableList()` |
| `getDirectionBreakdown(rows, name, type)` | `(rows, string, string) → [string, number][]` | Суммы по направлениям для статьи |

## Что НЕ меняется

- Вкладка «Таблица» (матрица по месяцам) — без изменений кроме порядка статей поступлений
- Выбор месяца через селектор в хедере
- Сайдбар-фильтры — применяются к «По статьям», не применяются к матрице
- Жёлтая плашка в матрице
