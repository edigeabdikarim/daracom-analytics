# ДДС: Редизайн вкладки «По статьям» в блоке «Куда ушли деньги»

**Дата:** 2026-03-22
**Файл:** `dds.html`
**Статус:** Approved

---

## Контекст

Текущая вкладка «По статьям» в блоке «Куда ушли деньги» показывает expandable-список статей с узкими барами (80px) и tooltip только по магазинам. Руководителю неудобно читать — данные «съезжают», непонятна структура расходов по направлениям бизнеса.

---

## Что меняется

### 1. Вкладка «По статьям» → таблица

Заменить expandable-список на таблицу со следующими колонками:

| Колонка | Описание |
|---------|----------|
| **Статья** | Название статьи |
| **Направления** | Мини-тэги направлений (Продажи, Зарплаты, Адм. расходы…), цвет `#F3F4F6` |
| **Доля** | Горизонтальный бар 110px. Расходы: градиент `#EF4444→#F97316`. Доходы: `#10B981→#34D399` |
| **Сумма, ₸** | Правое выравнивание. Расходы — красный `#EF4444`, доходы — зелёный `#10B981` |
| **%** | Двустрочная ячейка (см. ниже) |

### 2. Порядок групп

**Поступления — всегда первым блоком**, расходы — вторым.

Заголовок-строка группы:
- Поступления: фон `#F0FDF4`, текст `#059669`
- Расходы: фон `#FEF2F2`, текст `#B91C1C`

### 3. Фиксированный порядок статей в Поступлениях

Жёсткий приоритет (применяется везде — «По статьям», матрица «Таблица»):

1. **Выручка от продаж** — всегда первая
2. **Возврат товаров** — всегда вторая
3. Остальные — по убыванию суммы

Реализация: константа `INCOME_ARTICLE_PRIORITY = ['Выручка от продаж', 'Возврат товаров']` в коде. Сортировка: приоритетные статьи в начало по индексу в массиве, остальные по `amount desc`.

### 4. Двустрочная ячейка «%»

```
32%          ← крупно, жирно — доля статьи внутри своей группы
от выр: 21%  ← мелко, серо — % от общей суммы Поступлений
```

- Для **расходов**: строка 1 = статья / итого расходов × 100. Строка 2 = статья / итого поступлений × 100, подпись «от выр: X%»
- Для **поступлений**: строка 1 = статья / итого поступлений × 100, цвет зелёный. Строка 2 = «от итого» (без числа — не нужно)

### 5. Hover-tooltip (два блока)

При наведении на строку таблицы показывать всплывающую карточку:

```
┌─────────────────────────────┐
│ ПО НАПРАВЛЕНИЯМ             │  ← секция 1
│  Зарплаты    ████░  2 960 000│
│  Адм. расходы ██░░  1 040 000│
├─────────────────────────────┤
│ ПО МАГАЗИНАМ                │  ← секция 2
│  Маг. Астана  ████░   820 000│
│  Маг. Есентай ███░░   640 000│
│  ...все магазины...          │
└─────────────────────────────┘
```

- Цвета сумм: расходы `#F87171`, доходы `#34D399`
- Мини-бары внутри tooltip: ширина пропорциональна максимуму в группе
- Позиционирование: справа от курсора, умный flip если выходит за экран
- Использовать существующие функции: `getStoreBreakdown()` для магазинов; добавить `getDirectionBreakdown()` для направлений

### 6. Убрать click-to-expand

Убрать: `toggleExpandArticle()`, `buildExpandBody()`, `.article-expand-body`, `.expand-chevron`, `expandedArticles`.
Оставить только hover-tooltip как основное взаимодействие.

### 7. Матрица «Таблица» — фиксированный порядок

В `renderMatrixTable()` применить тот же `INCOME_ARTICLE_PRIORITY` для сортировки поступлений.

---

## Что НЕ меняется

- Вкладка «Таблица» (матрица по месяцам) — без изменений кроме порядка статей
- Выбор месяца через селектор в хедере — работает как прежде
- Сайдбар-фильтры (магазин, направление, тип) — применяются к «По статьям», не применяются к матрице
- Жёлтая плашка в матрице — остаётся

---

## Новые функции

| Функция | Назначение |
|---------|-----------|
| `getDirectionBreakdown(rows, name, type)` | Суммы по направлениям для статьи (аналог `getStoreBreakdown`) |
| `sortIncomeArticles(articles)` | Применяет `INCOME_ARTICLE_PRIORITY`, остальные по сумме |
| `renderArticlesTable()` | Новый рендер таблицы, заменяет `renderExpandableList()` |

---

## Затронутые функции (изменить)

| Функция | Что делать |
|---------|-----------|
| `renderExpandableList()` | Заменить на `renderArticlesTable()` |
| `renderMatrixTable()` | Добавить `sortIncomeArticles()` для поступлений |
| `showTooltip()` | Добавить блок «По направлениям» перед «По магазинам» |
| CSS `.article-*` | Убрать стили expand-body, chevron; добавить стили таблицы |
