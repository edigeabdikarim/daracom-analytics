# Spec: Улучшения карточек «По магазинам»

**Дата:** 2026-03-30
**Файл:** `dds.html` — функция `renderStoreGrid()` + `@media print`

---

## Цель

Устранить некорректный учёт возвратов в выбытиях торговых карточек, добавить карточки Компания/Офис с полноценной разбивкой поступлений и расходов, улучшить читаемость таблицы групп и качество печати раздела «По магазинам».

---

## 1. Разделение логики: магазины vs офисные карточки

Ввести константу:
```js
const OFFICE_STORES = new Set(['Компания', 'Офис']);
```

Карточки делятся на два типа:
- **Торговые** (не в `OFFICE_STORES`): Астана, Мира, Есентай, Абайка, Восход, Above, Above Астана, Kaspi
- **Офисные** (`OFFICE_STORES`): Компания, Офис

«Компания» возвращается в гриду — убрать исключение `s !== 'Компания'` в фильтре сортировки магазинов.

## 2. Агрегация данных

### storeMap — расширение структуры
```js
storeMap[store] = {
  inc: 0, out: 0,
  salesInc: 0, salesOut: 0,
  groups: {},      // расходные группы (для breakdown выбытий)
  incGroups: {}    // доходные группы (только для OFFICE_STORES)
}
```

### Торговые карточки
- `d.out` — **исключать** строки с артикулом «Возвраты товаров» (группа «Продажи», тип «Выбытие»). Обоснование: возвраты уже учтены в блоке Продажи как `salesOut`, двойной учёт искажает Выбытия.
- Breakdown расходов: применять `EXCLUDED_STORE_GROUPS` как прежде (`['Платежи', 'Основные средства', 'Финансирование', 'Продажи']`).

### Офисные карточки (Компания, Офис)
- `d.out` — все выбытия без исключений.
- `incGroups`: агрегировать поступления по группе статьи (`ARTICLE_GROUP[r.article]`), исключая группу «Продажи». Заполняется при `r.type === 'Поступление'`.
- Breakdown расходов: **без** фильтра `EXCLUDED_STORE_GROUPS` — включать все группы.

## 3. Рендер карточки

### Торговая карточка (без изменений в структуре)
```
[Название]
Продажи       X ₸  (зелёный)
Выбытия       X ₸  (красный, без Возвратов товаров)
──────────────────────
Чистый CF     ±X ₸
──────────────────────
Затраты от продаж   XX%  [бар]
ВЫБЫТИЯ ПО ГРУППАМ  (таблица)
  Группа А    X ₸   Y%
  Группа Б    X ₸   Y%
```
- % в таблице групп = `amt / salesTotal * 100` — **сохраняется**.

### Офисная карточка (новый вид)
```
[Название]
Выбытия       X ₸  (красный)
──────────────────────
Чистый CF     ±X ₸
──────────────────────
ПОСТУПЛЕНИЯ ПО ГРУППАМ  (таблица, если есть)
  Группа А    X ₸
  Группа Б    X ₸
──────────────────────
ВЫБЫТИЯ ПО ГРУППАМ  (таблица)
  Группа А    X ₸
  Группа Б    X ₸
```
- Строки «Продажи» и «Затраты от продаж» / бар — **не показываются**.
- В таблицах групп — **без %** (нет релевантной базы).
- Блоки поступлений/выбытий показываются только если соответствующие данные не пусты.

## 4. Таблица «Выбытия по группам» (все карточки)

Заменить текущие `<div>` строки на `<table>`:

```html
<table style="width:100%;border-collapse:collapse;font-size:11px;">
  <tr>
    <td style="padding:2px 4px;color:#374151;border-bottom:1px solid #E5E7EB;">Группа</td>
    <td style="padding:2px 4px;color:#EF4444;font-weight:600;text-align:right;border-bottom:1px solid #E5E7EB;">X ₸</td>
    <!-- для торговых: -->
    <td style="padding:2px 4px;color:#6B7280;font-weight:700;text-align:right;border-bottom:1px solid #E5E7EB;">Y%</td>
  </tr>
</table>
```

- Убрать мини-бар (плохо читается при печати, нет ценности в таблице).
- Разделители строк через `border-bottom: 1px solid #E5E7EB` — видны при печати.

## 5. Печать (`@media print`)

### 5.1 Контрастный шрифт
```css
#storeGrid .store-card { color: #111 !important; }
#storeGrid .store-card * { color: inherit; }
```
Цветные значения (CF, продажи, выбытия) переопределяются явными inline-стилями — они сохраняются через `print-color-adjust: exact`.

### 5.2 Остатки на начало/конец месяца
Добавить скрытый блок `#storesPrintHeader` перед `#storeGrid`:
```html
<div id="storesPrintHeader" style="display:none;margin-bottom:16px;">
  <span>Остаток на начало месяца: <b id="sprintBalOpen">—</b></span>
  <span style="margin-left:24px;">Остаток на конец месяца: <b id="sprintBalClose">—</b></span>
</div>
```
Заполняется в `renderStoreGrid()` теми же значениями, что используются в KPI (`balanceOpening`, `balanceClosing`).
В `@media print`: `#storesPrintHeader { display: block !important; }`

### 5.3 Без обрезания карточек
```css
.store-card { page-break-inside: avoid; break-inside: avoid; }
```
