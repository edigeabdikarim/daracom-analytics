# Спека: Разбивка по кошелькам в разделе Обзор

**Дата:** 2026-03-31  
**Статус:** Approved

---

## Цель

Добавить в раздел «Обзор» (dds.html) детализацию остатков по кошелькам под существующими 6 KPI-карточками. Руководство должно видеть, сколько денег есть наличными и безналичными по каждому направлению бизнеса.

---

## Источник данных

Лист **«ДДС: Сводный»** в Google Sheets (`DDS_SHEET_ID`):

- Строка 1: заголовок (месяцы 1–12 в колонках B–M)
- Строка 2: «Денег на начало месяца» — суммарный баланс (уже используется)
- Строки 3–26: остатки по отдельным кошелькам за каждый месяц

Значение в колонке N для месяца N = **остаток на начало месяца N** (= конец месяца N-1).  
Для текущего месяца колонка содержит **актуальный остаток** (обновляется вручную своевременно).

## Логика отображения

### Определение «текущий месяц»

Месяц считается **текущим**, если для него нет загруженных транзакций следующего месяца (`filteredRows` пустые для `currentMonth + 1`). Практически: `currentMonth` равен максимальному месяцу с данными в `allRows`.

### Прошедший месяц

Каждая строка кошелька показывает два значения:

```
Мира     33 100  →  158 123
```

- Левое: остаток на **начало** месяца = `wallets[month][key]`
- Правое: остаток на **конец** месяца = `wallets[month + 1][key]`

### Текущий месяц

Каждая строка показывает одно значение — **актуальный остаток**:

```
Мира     158 123
```

Метка «Актуально» или зелёная точка рядом с заголовком блока.

### 3. `dds.html` — новая функция `renderWallets()`

Вызывается из `renderKPI()` после рендера 6 карточек. Принимает `currentMonth` и `monthlyBalance`. Определяет режим (текущий / прошедший), рендерит две колонки с итогами.

Обновить `BALANCE_VERSION` (например `v4-with-wallets`) чтобы старый кеш без поля `wallets` был инвалидирован при первом открытии.
