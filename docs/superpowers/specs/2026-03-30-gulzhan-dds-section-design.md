# Дизайн: раздел «ДДС Гульжан А.» в dds.html

**Дата:** 2026-03-30
**Статус:** Утверждён

---

## Назначение

Новый раздел в `dds.html` — дубликат «Сводной», но с учётом реального периода расходов:
- Аренда оплачивается **наперёд** (январский платёж покрывает февраль) → показываем данные M-1
- Зарплаты и Налоги ЗП оплачиваются **за предыдущий месяц** (февральский платёж = январская зарплата) → показываем данные M+1
- Все остальные статьи — данные текущего месяца M (как в обычной Сводной)

---

## Навигация

Порядок табов:
```
Обзор | Куда ушли | По магазинам | Сводная | ДДС Гульжан А.
```
- `id="nav-gulzhan"` / `id="section-gulzhan"`
- Активируется через `setSection('gulzhan')`

### Загрузка смежных месяцев

При открытии раздела (и при смене месяца) вызываем `ensureMonthLoaded(M-1)` и `ensureMonthLoaded(M+1)`.

`ensureMonthLoaded(m)` — выделяется как утилита из существующей логики `loadAllMonths()`:
- Если `m < 1` или `m > 12` — пропускаем
- Если `allMonthsLoaded[m]` уже `'done'` — пропускаем
- Иначе загружаем через GAS `?action=dds&month=m` и сохраняем в `allMonthsData[m]`

Пока данные грузятся — в ячейках статей с зависимостью показываем `…` (спиннер-заглушка).

### Источники данных для `buildGulzhanData(month)`

| Статьи | Источник данных |
|--------|----------------|
| Аренда торговых точек, Аренда офиса, Аренда и Охрана Склада | `allMonthsData[month-1]` или `GULZHAN_DEC2025_RENT` если `month === 1` |
| Зарплата × 6 статей + Налоги ЗП | `allMonthsData[month+1]` или прочерк (`—`) если данных нет |
| Все остальные статьи | `allMonthsData[month]` (= `allRows`) |

## Секция 2: Классификация статей

```js
// Оплачиваются наперёд (данные из M-1)
const GULZHAN_ADVANCE_ARTICLES = new Set([
  'Аренда торговых точек',
  'Аренда офиса',
  'Аренда и Охрана Склада',
]);

// Оплачиваются за предыдущий месяц (данные из M+1)
const GULZHAN_PREV_MONTH_ARTICLES = new Set([
  'Зарплата Продавцы',
  'Зарплата Офис',
  'Зарплата Аутсорс',
  'Зарплата Техперсонал',
  'Зарплата СММ маркетинг, мерчандайзер',
  'Зарплата Водитель',
  'Налоги ЗП',
]);
```

### Подсветка

| Тип | CSS-класс | Фон | Цвет текста |
|-----|-----------|-----|------------|
| Оплата наперёд | `gul-advance` | `#FEF3C7` | `#92400E` |
| Оплата за пред. месяц | `gul-prevmonth` | `#EDE9FE` | `#5B21B6` |

Класс применяется к `<td>` с именем статьи.

### Подпись под названием статьи

- Аренда (advance): `▲ данные за [Пред. месяц]`
- Зарплаты/НалогиЗП (prevmonth): `▼ данные за [След. месяц]`

Пример рендера ячейки:
```html
<td class="gul-advance">
  Аренда торговых точек
  <div class="gul-shift-label">▲ данные за Февраль</div>
</td>
```

## Секция 3: Интерфейс и рендер

### Шапка раздела

Аналогична Сводной:
- Заголовок «ДДС Гульжан А.»
- Чипы остатков (Нач. / Кон. месяца) — `#gulzhanBalanceChips`
- Чекбокс «Скрыть нулевые» — `#gulzhanHideZeros`
- Кнопка «Печать»
- Легенда:
  ```
  🟡 Оплата наперёд (данные пред. месяца)   🟣 Оплата за пред. месяц (данные след. месяца)
  ```

### Таблица

Структура идентична Сводной: статьи × магазины + Итого + % выр.
Отличия только в:
1. Источнике данных для сдвинутых статей (секция 1)
2. CSS-классе + лейбле на ячейке имени статьи (секция 2)

### Функция рендера

`renderGulzhanMatrix()` — вызывается при:
- `setSection('gulzhan')`
- Смене месяца в сайдбаре (если активен раздел gulzhan)
- Завершении загрузки смежного месяца

### Печать

- Печатный заголовок: `#gulzhanPrintHeader` (название + месяц + остатки)
- Те же `@media print` стили что у Сводной

## Граничные случаи

| Ситуация | Поведение |
|----------|-----------|
| `month === 1`, аренда нет M-1 | Использовать `GULZHAN_DEC2025_RENT` |
| M+1 не загружен ещё | Показывать `…` пока грузится, затем обновить |
| M+1 загружен, но нет данных (не оплачено) | Показывать `—` |
